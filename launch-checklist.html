🚨 Critical Blockers — Must Fix Before Launch
1. Missing Stripe Price IDs (Yearly + Enterprise)
CRITICAL

4 yearly price IDs and the Enterprise monthly ID are fake placeholder strings. Any user who clicks "Yearly" or tries to buy Enterprise will get a Stripe error.

src/lib/subscriptionTiers.ts — price_starter_yearly, price_pro_yearly, price_enterprise_yearly, price_enterprise

✏️ Fix: Go to Stripe Dashboard → Products → create yearly prices for Starter ($290/yr) and Pro ($370/yr), create Enterprise plan. Copy real price_1xxx IDs into subscriptionTiers.ts.
2. No Stripe Webhook Handler
CRITICAL

When a customer cancels, downgrades, or has a failed payment in Stripe, your app never finds out. Their subscription stays "active" until the next manual check-subscription call.

✏️ Fix: Create supabase/functions/stripe-webhook/index.ts that handles customer.subscription.deleted, invoice.payment_failed, and customer.subscription.updated events. Register the webhook URL in Stripe Dashboard.
3. Missing API Keys in Supabase Edge Functions
CRITICAL

Without these keys set in Supabase → Settings → Edge Functions → Secrets, every email, AI feature, and checkout will fail in production.

✏️ Fix: In Supabase Dashboard → Edge Functions → Secrets, add:
STRIPE_SECRET_KEY = sk_live_...
RESEND_API_KEY = re_...
LOVABLE_API_KEY = (your key for AI analysis functions)
4. AI Document Analysis May Not Work (7 Edge Functions)
CRITICAL

7 functions (analyze-receipt, analyze-bank-statement, analyze-tax-document, categorize-transaction, forecast-cashflow, schedule-c-analyze, tax-qa) call the Lovable AI gateway. These need a valid LOVABLE_API_KEY set in Supabase secrets.

✏️ Fix: Set LOVABLE_API_KEY in Supabase secrets. Test each function by uploading a real document and checking logs in Supabase Dashboard → Edge Functions → Logs.
5. Email Domain Not Verified (All Emails Will Bounce)
CRITICAL

Emails send from support@majrtaxsoftware.com and support@prm3tax.com. If neither domain has DKIM/SPF records configured in Resend, all invoices, invitations, and password resets bounce or go to spam.

✏️ Fix: Log in to Resend.com → Domains → Add your domain → Add the DNS records they provide → Verify. Then test sending a real invoice email end-to-end.
6. Bank Linking Shows Fake Mock Accounts
CRITICAL

BankLinkingFlow.tsx contains a hardcoded mockAccounts array with fake Checking/Savings/Credit Card accounts. Users "link" their bank and see fake data — this is a serious trust issue.

src/components/BankLinkingFlow.tsx — mockAccounts array

✏️ Fix: Either integrate Plaid (plaid.com — industry standard bank API) or hide the Bank Linking feature entirely until it is real. Do not ship fake bank data.
7. Supabase Database Migrations Need to Run on Production
CRITICAL

65+ migration files exist locally but may not all be applied to the live Supabase project. Any missing table or column causes silent insert failures. RLS policies may not be enabled on all tables.

✏️ Fix: Run `supabase db push` or apply migrations via Supabase Dashboard → SQL Editor. Then test that clients, invoices, transactions, and employees tables all work with RLS enabled.
8. Enterprise Price ID Is Placeholder (Blocks Enterprise Sales)
CRITICAL

prod_enterprise and price_enterprise are fake. Any attempt to subscribe to Enterprise sends an invalid ID to Stripe and fails.

✏️ Fix: Create Enterprise product in Stripe → copy real price ID → update subscriptionTiers.ts.

Yearly Billing Toggle Does Nothing (Placeholder IDs)
IMPORTANT

The monthly/yearly toggle in UpgradeModal shows discounted yearly prices but all yearly price IDs are placeholders. A warning banner was added (recent fix) but yearly billing is not actually functional.

✏️ Fix: Create yearly prices in Stripe, replace placeholder IDs. Depends on Critical #1.
BillingDetails.tsx Has 100% Mock Data
IMPORTANT

The Billing Details page shows fake payment methods, fake billing address, and fake invoice history. Users visiting this page see data that isn't real.

✏️ Fix: Either connect to Stripe Customer Portal for real billing management (easier) or hide the page and redirect to the Stripe portal instead.
Recurring Invoices / Tasks Not Scheduled
IMPORTANT

Edge functions generate-recurring-invoices and generate-recurring-tasks exist but are never triggered automatically. Recurring items only generate if called manually.

✏️ Fix: Set up a Supabase cron job (pg_cron) or GitHub Actions to call these functions daily.
Client Portal Route Guard — Clients Can Access Admin Routes
IMPORTANT

ClientRouteGuard.tsx gives full access to all routes for trial/subscribed users regardless of role. A client user could navigate directly to /admin, /team, /payroll-runs if they know the URL.

✏️ Fix: Add role-based route guards — clients should only access /client-* routes. Admin routes should require isAdmin. User routes should require role=user or admin.
"Coming Soon" Buttons Scattered Throughout UI
IMPORTANT

Multiple buttons and sections are disabled with "Coming Soon" labels: Change Password in ClientSettings, Download invoices in ClientInvoices, Yearly billing in UpgradeModal. Users see broken expectations.

✏️ Fix: Either implement these features or remove them entirely from the UI before launch.
No Rate Limiting on Email Edge Functions
IMPORTANT

send-invoice, send-client-invitation, and other email functions have no per-user rate limits. A malicious or buggy user could spam thousands of emails and exhaust your Resend quota.

✏️ Fix: Add a check in each email function: verify user has not sent more than 20 emails in the past hour using a simple counter in Supabase.
FirmSubscriptions Seat Management Is Frontend-Only
IMPORTANT

The "Manage Seats" dialog updates local React state only — the seat count is not saved to the database or Stripe. Every page refresh resets the count.

✏️ Fix: Either persist seat count to a Supabase table, or remove the Manage Seats button until it is implemented.
setup-storage Edge Function Must Run Before First Upload
IMPORTANT

The bank-statements and client-documents storage buckets are created on-demand by setup-storage. If this function fails (missing env vars), uploads fail silently.

✏️ Fix: Manually create both buckets in Supabase Dashboard → Storage → New bucket (set to private). This removes the dependency on the edge function for bucket creation.
Password Change Not Implemented in Client Settings
IMPORTANT

ClientSettings.tsx has a "Change Password" button that is permanently disabled with "Coming Soon". Clients have no way to change their password through the portal.

✏️ Fix: Use supabase.auth.updateUser({ password: newPassword }) — it is a one-liner. Or send a password reset email instead.
Invoice Download Button Disabled in Client Portal
IMPORTANT

ClientInvoices.tsx has a Download button per invoice that is disabled and shows "Download coming soon". Clients cannot download their own invoices.

✏️ Fix: Use the existing jsPDF setup (BillingDetails already imports it) to generate a simple invoice PDF on click.
